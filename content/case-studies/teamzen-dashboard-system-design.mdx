## Overview

TeamZen focused on an internal operations use case where users need to move quickly between status checking, record review, and next action. The main challenge was balancing visibility and cognitive load.

## Problem Framing

Operations dashboards often fail by being comprehensive but not usable.

## Design Decision

I prioritized:

- High-signal summary metrics first
- Contextual detail only when needed
- Stable layout zones to reduce cognitive switching

## Practical Outcome

- Teams can scan top-level status before entering detailed tables.
- Repetitive navigation drops when section zones stay consistent.
- Action pathways become clearer when summaries map directly to follow-up views.
